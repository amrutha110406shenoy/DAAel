🧠 Breaking the O(N²) Matrix Trap

Legacy DNA compressors like GenCompress find approximate repeats by filling an O(N²) edit-distance matrix — comparing every position against every other. For a genome of 1M bases, that's 1 trillion comparisons.

CAGE replaces this with LSH bucket routing — similar blocks are hashed into shared buckets in O(1) per lookup, giving overall O(N log N) with a strict O(N) memory bound.

🔄 Reverse-Complement (RC) Awareness

DNA is double-stranded. A segment may repeat in reverse-complement form elsewhere (e.g., ACGT matching ACGT's RC = ACGT). Standard compressors miss these.

CAGE solves this with Canonical MinHash: for every k-mer, it picks min(kmer, RC(kmer)) lexicographically. This makes the sketch identical for both strands — so RC-related blocks automatically land in the same bucket.

🧬 Comprehensive Algorithm Comparison Window

DNACompress
Uses PatternHunter to find exact/approximate repeats, then encodes non-repeats using a 2-bit or Huffman model.
✅ Best For:
Large sequences where heuristic suffix tree approximations provide a good balance of speed and ratio.
Time: O(N log N) Space: O(N)
BioCompress / BioCompress-2
Sliding window (Ziv-Lempel style) finding longest exact repeats. BioCompress-2 adds 2nd order Markov models.
✅ Best For:
Small, simple sequences with long, identical exact repeats. Struggles with mutations.
Time: O(N²) Space: O(N)
GenCompress
One-pass algorithm using dynamic programming to find optimal approximate repeats with edit operations.
✅ Best For:
Moderate-sized sequences where getting the absolute maximal compression ratio is worth the long computation time.
Time: O(N²) Space: O(N²)
RC-Aware CAGE (Proposed)
Concurrent MinHash LSH chunking mapping mutations and reverse-complements in O(1) bucket lookups.
✅ Best For:
Extremely large, highly mutated genomes with structural variants and reverse-complements. Ultra-fast probabilistic approximation.
Time: O(N log N) Space: O(N)
